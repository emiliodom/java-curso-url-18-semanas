Referencia rápida — Condicionales, ciclos, métodos y clases

Tenla abierta durante ejercicios, challenge y la Guía 2.

1. Condicionales

if (condicion) {
  // se ejecuta si condicion es true
} else if (otraCondicion) {
  // se ejecuta si la anterior fue false y esta true
} else {
  // en cualquier otro caso
}
switch (variable) {
  case valor1:
    // ...
    break;          // ¡no lo olvides!
  case valor2:
    // ...
    break;
  default:
    // ...
}
Comparar String: usa texto.equals("algo"), nunca texto == "algo".

2. Ciclos

// Cuando SÉ cuántas vueltas
for (int i = 0; i < 5; i++) {
  // se repite 5 veces: i = 0,1,2,3,4
}
// Cuando NO sé cuántas vueltas
while (condicion) {
  // se repite mientras condicion sea true
  // debe existir algo que la haga false eventualmente
}
// Ejecuta AL MENOS una vez
do {
  // ...
} while (condicion);
⚠ Ciclo infinito: si la condición nunca se vuelve falsa (olvidaste el i++ o la resta), el programa se cuelga.

3. Métodos

static TIPO nombreMetodo(TIPO parametro1, TIPO parametro2) {
  // ...
  return valor;   // obligatorio si TIPO no es void
}

static void otroMetodo() {
  // no retorna nada
}
ParteSignifica
staticSe puede llamar directo desde main sin crear un objeto
void / int / boolean…Tipo de dato que retorna (void = nada)
(int a, int b)Parámetros: datos de entrada del método
return valor;Entrega el resultado y termina el método

4. Clases y objetos (primeros pasos)

class Mascota {
  String nombre;    // campo
  int edad;         // campo

  void hacerSonido() {   // método de la clase
    System.out.println(nombre + ": ¡Guau!");
  }
}

// En main:
Mascota firulais = new Mascota();  // crear objeto
firulais.nombre = "Firulais";      // acceder a un campo
firulais.hacerSonido();            // llamar un método
Clase = molde/plano. Objeto = algo creado a partir del molde con new. Cada objeto tiene sus propios valores.

5. Convenciones (repaso + nuevo)

  • Variables y métodos: camelCase — calcularPromedio, esAprobatorio.
  • Métodos booleanos: nómbralos como pregunta — esPar, puedeVotar.
  • Clases: PascalCase (primera letra mayúscula) — Mascota, Beneficiario.
  • Indenta cada bloque { } — si no ves la "escalera", revisa el anidado.

6. Errores comunes de esta semana

El compilador dice / síntoma…Significa…
El switch ejecuta más casos de los esperadosOlvidaste un break ("fallthrough").
El programa nunca termina (se cuelga)Ciclo infinito: la condición del for/while nunca se vuelve falsa.
missing return statementUn método con tipo de retorno (no void) tiene un camino sin return.
cannot find symbol (con un método)Llamaste el método con otro nombre/mayúsculas, o con distinta cantidad de parámetros.
Comparación de texto rara con ==Compara String con .equals(), no ==.
class X is public, should be in X.javaSi declaras varias clases, solo UNA puede ser public y su nombre debe coincidir con el archivo.
