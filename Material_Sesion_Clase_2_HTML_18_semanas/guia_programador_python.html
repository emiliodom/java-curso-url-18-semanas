De Python a Java: condicionales, ciclos, funciones y clases

Segunda parada del puente Python↔Java. Hoy comparamos cómo cada lenguaje decide, repite, organiza en funciones/métodos y da sus primeros pasos hacia los objetos.
Recordatorio de la Semana 1
Java exige declarar tipos y usa ; y { } donde Python usa indentación. Hoy verás que la lógica (cómo piensas el problema) es idéntica entre ambos lenguajes — solo cambia la sintaxis.

1. Condicionales: if/elif/else vs if/else if/else

🐍 Python☕ Java
 
edad = 16

if edad >= 18:
    print("Puedes votar")
elif edad >= 16:
    print("Voto parcial")
else:
    print("Aún no")
 
int edad = 16;

if (edad >= 18) {
  System.out.println("Puedes votar");
} else if (edad >= 16) {
  System.out.println("Voto parcial");
} else {
  System.out.println("Aún no");
}
Diferencias: elif → else if, dos puntos : → llaves { }, e indentación obligatoria → llaves obligatorias (la indentación en Java es solo buena práctica, no la exige el compilador).

2. El switch — Python no lo tuvo por años

🐍 Python (match, desde 3.10)☕ Java (switch, desde siempre)
 
match dia:
    case 1:
        nombre = "Lunes"
    case 2:
        nombre = "Martes"
    case _:
        nombre = "Inválido"
 
switch (dia) {
  case 1:
    nombre = "Lunes";
    break;
  case 2:
    nombre = "Martes";
    break;
  default:
    nombre = "Inválido";
}
Antes de Python 3.10, la alternativa típica era un diccionario o una cadena de if/elif. Ojo: Java exige break en el switch clásico o "cae" al siguiente caso — Python's match no tiene ese problema.

3. Ciclos: for-in-range vs for clásico

🐍 Python☕ Java
 
for i in range(5):
    print(i)
# imprime 0,1,2,3,4
 
for (int i = 0; i < 5; i++) {
  System.out.println(i);
}
// imprime 0,1,2,3,4
 
i = 100
while i > 0:
    i -= 15
    print(i)
 
int i = 100;
while (i > 0) {
  i -= 15;
  System.out.println(i);
}
range(5) ya calcula "5 vueltas" por ti; en Java escribes las tres partes del for a mano (inicio, condición, incremento) — más verboso, pero más explícito sobre qué está pasando.

4. Funciones (def) vs métodos (static)

🐍 Python☕ Java
 
def convertir_celsius_a_fahrenheit(celsius):
    return celsius * 9 / 5 + 32

fahrenheit = convertir_celsius_a_fahrenheit(30)
print(fahrenheit)
 
static double convertirCelsiusAFahrenheit(double celsius) {
  return celsius * 9 / 5 + 32;
}

double fahrenheit = convertirCelsiusAFahrenheit(30);
System.out.println(fahrenheit);
Python no exige declarar el tipo de los parámetros ni del retorno (aunque puedes usar "type hints" opcionales); Java sí, siempre. Nombre: en Python se usa snake_case (convertir_celsius_a_fahrenheit); en Java, camelCase (convertirCelsiusAFahrenheit).

5. Clases: Python (self, __init__) vs Java (this, new)

🐍 Python☕ Java
 
class Libro:
    def __init__(self, titulo, autor):
        self.titulo = titulo
        self.autor = autor

    def mostrar_info(self):
        print(f"{self.titulo} — {self.autor}")

quijote = Libro("Don Quijote", "Cervantes")
quijote.mostrar_info()
 
class Libro {
  String titulo;
  String autor;

  void mostrarInfo() {
    System.out.println(titulo + " — " + autor);
  }
}

Libro quijote = new Libro();
quijote.titulo = "Don Quijote";
quijote.autor = "Cervantes";
quijote.mostrarInfo();
Hoy en Java usamos la forma más simple (asignar los campos directo, sin constructor). Python casi siempre usa __init__ (equivalente a un "constructor") desde el día 1; en Java verás constructores más adelante en el curso — por ahora, campos públicos y asignación directa. self en Python ≈ this en Java (ambos se refieren "al propio objeto"), aunque hoy no lo necesitamos explícitamente.

7. Ejercicios de traducción (Python → Java)

Traducción 1 — Clasificar una donación

monto = 250

if monto >= 500:
    print("Donación mayor")
elif monto >= 100:
    print("Donación media")
else:
    print("Donación básica")
Solución 
public class Main {
  public static void main(String[] args) {
    double monto = 250;

    if (monto >= 500) {
      System.out.println("Donación mayor");
    } else if (monto >= 100) {
      System.out.println("Donación media");
    } else {
      System.out.println("Donación básica");
    }
  }
}
Aquí no necesitas .equals(): comparar números con >= funciona igual en ambos lenguajes. Ese cuidado especial es solo para texto (String) — como en el Ejercicio 1 de ejercicios.html, que compara colores en vez de montos.

Traducción 2 — Ciclo con función

def es_multiplo_de_tres(numero):
    return numero % 3 == 0

for i in range(1, 11):
    print(i, es_multiplo_de_tres(i))
Solución 
public class Main {
  public static void main(String[] args) {
    for (int i = 1; i < 11; i++) {
      System.out.println(i + " " + esMultiploDeTres(i));
    }
  }

  static boolean esMultiploDeTres(int numero) {
    return numero % 3 == 0;
  }
}
Misma idea que el esPar del Ejercicio 6 (ejercicios.html): un método booleano probado dentro de un ciclo, aquí con %3 en vez de %2.

Traducción 3 — Clase simple

class Libro:
    def __init__(self, titulo):
        self.titulo = titulo

    def mostrar_titulo(self):
        print("Libro: " + self.titulo)

l = Libro("Cien años de soledad")
l.mostrar_titulo()
Solución 
public class Main {
  public static void main(String[] args) {
    Libro l = new Libro();
    l.titulo = "Cien años de soledad";
    l.mostrarTitulo();
  }
}

class Libro {
  String titulo;

  void mostrarTitulo() {
    System.out.println("Libro: " + titulo);
  }
}
Versión reducida (un solo campo) de la clase Libro de la sección 5. Compárala con la clase Mascota del Ejercicio 7 (ejercicios.html): misma idea, otro dominio.

Traducción 4 — Reto inverso (Java → Python)

¿Qué imprime este Java? Tradúcelo mentalmente a Python:

public class Main {
  public static void main(String[] args) {
    int total = 0;
    for (int i = 1; i <= 5; i++) {
      total += i;
    }
    System.out.println(total);
  }
}
Solución
Imprime 15 (suma 1+2+3+4+5). En Python:
total = 0
for i in range(1, 6):
    total += i
print(total)  # 15
