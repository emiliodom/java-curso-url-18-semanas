Normas rápidas

  • Todo bloque { } se indenta: si no ves la "escalera", algo está mal anidado.
  • En switch clásico, no olvides el break (o cae al siguiente caso).
  • Todo for/while necesita una condición que eventualmente sea falsa: si no, ciclo infinito.
  • Nombre de métodos en camelCase y como verbos: calcularPromedio, esMayorDeEdad.

Pregunta detonadora: "¿cómo decide un semáforo qué luz encender?" → puente a: condición = pregunta con respuesta true/false (repaso de operadores de comparación de la Semana 1).

public class Main {
  public static void main(String[] args) {
    int edad = 16;

    if (edad >= 18) {
      System.out.println("Puedes votar");
    } else if (edad >= 16) {
      System.out.println("Puedes votar solo en algunas elecciones");
    } else {
      System.out.println("Aún no puedes votar");
    }
  }
}
  • El programa evalúa de arriba hacia abajo y ejecuta el primer bloque cuya condición sea verdadera; el resto se ignora.
  • else es el "en cualquier otro caso" — nunca lleva condición propia.

Ahora el mismo tipo de decisión con switch (útil cuando comparas UNA variable contra varios valores exactos):

public class Main {
  public static void main(String[] args) {
    int diaSemana = 3;
    String nombreDia;

    switch (diaSemana) {
      case 1:
        nombreDia = "Lunes";
        break;
      case 2:
        nombreDia = "Martes";
        break;
      case 3:
        nombreDia = "Miércoles";
        break;
      default:
        nombreDia = "Día inválido";
    }

    System.out.println(nombreDia);
  }
}
Demostración del error (¡a propósito!)
Borra un break y ejecuta: el programa sigue ejecutando el case siguiente ("fallthrough"). Es uno de los errores silenciosos más comunes de switch.
Bonus: sintaxis moderna (opcional) 
String nombreDia = switch (diaSemana) {
  case 1 -> "Lunes";
  case 2 -> "Martes";
  case 3 -> "Miércoles";
  default -> "Día inválido";
};
Java moderno permite switch como expresión con -> (sin break, sin fallthrough). Se acepta cualquiera de las dos formas en este curso; verás esta variante en más código de terceros.

Metáfora: un ciclo es una instrucción que se repite mientras una condición sea verdadera. for cuando sabes cuántas veces; while cuando no lo sabes de antemano.

public class Main {
  public static void main(String[] args) {
    // for: sabemos que son exactamente 5 vueltas
    for (int i = 1; i <= 5; i++) {
      System.out.println("Vuelta número: " + i);
    }

    // while: repetimos hasta que se cumpla una condición
    int saldo = 1000;
    int mes = 0;
    while (saldo > 0) {
      saldo = saldo - 250;
      mes++;
      System.out.println("Mes " + mes + " — saldo restante: " + saldo);
    }
  }
}
  • for (inicio; condición; incremento): las tres partes van separadas por ;. Si omites el incremento (i++), el ciclo nunca termina.
  • while: revisa la condición antes de cada vuelta. Si la condición nace falsa, el bloque nunca se ejecuta.
Demostración del ciclo infinito (¡a propósito, con cuidado!)
Cambia i++ por nada (quítalo) y explica qué pasaría (el compilador online se colgará: hay que detenerlo/recargar la pestaña). Mensaje clave: todo ciclo necesita una salida garantizada.
Preguntas rápidas
  • ¿Cuántas veces imprime for (int i = 0; i < 5; i++)? (Respuesta: 5 veces, i = 0,1,2,3,4 — ¡ojo con el "off-by-one"!)
  • ¿Qué diferencia hay entre i < 5 e i <= 5?

Metáfora: un método es una "mini receta" reutilizable. Le das ingredientes (parámetros) y a veces te devuelve un resultado (retorno).

public class Main {
  public static void main(String[] args) {
    // Método sin retorno (void): solo hace una acción
    saludar("Ana");

    // Método CON retorno: produce un valor que puedo usar
    int promedio = calcularPromedio(80, 90, 70);
    System.out.println("Promedio: " + promedio);

    boolean aprobado = esAprobatorio(promedio);
    System.out.println("¿Aprobado?: " + aprobado);
  }

  static void saludar(String nombre) {
    System.out.println("Hola, " + nombre + "!");
  }

  static int calcularPromedio(int a, int b, int c) {
    return (a + b + c) / 3;
  }

  static boolean esAprobatorio(int promedio) {
    return promedio >= 61;
  }
}
  • void = "no devuelve nada"; cualquier otro tipo (int, boolean, String…) exige un return.
  • Los parámetros son variables locales al método: a, b, c solo existen dentro de calcularPromedio.
  • El valor que return entrega puede guardarse en una variable (como promedio) o usarse directamente.

Hasta ahora todo vivía "suelto" dentro de Main. Hoy damos el primer paso de Programación Orientada a Objetos: agrupar datos relacionados en una clase y crear objetos a partir de ella.

public class Main {
  public static void main(String[] args) {
    // Creamos DOS objetos a partir de la misma clase Mascota
    Mascota firulais = new Mascota();
    firulais.nombre = "Firulais";
    firulais.edad = 3;

    Mascota michi = new Mascota();
    michi.nombre = "Michi";
    michi.edad = 1;

    firulais.hacerSonido();
    michi.hacerSonido();
  }
}

class Mascota {
  String nombre;
  int edad;

  void hacerSonido() {
    System.out.println(nombre + " (edad " + edad + ") hace: ¡Guau!");
  }
}
  • Una clase (Mascota) es un molde: describe qué datos (nombre, edad) y qué acciones (hacerSonido()) tendrá cada objeto.
  • Un objeto (firulais, michi) se crea con new NombreClase(). Cada objeto tiene sus propios valores.
  • Se accede a los datos y acciones de un objeto con el punto: firulais.nombre, firulais.hacerSonido().
  • Conexión con el proyecto del curso: en la Semana 12 (persistencia), cada "entidad" (beneficiario, usuario) será exactamente esto: una clase con campos y objetos guardados en la base de datos.
