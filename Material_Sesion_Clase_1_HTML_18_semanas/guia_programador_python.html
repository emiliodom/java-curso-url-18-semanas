La Guía del Programador de Python · Semana 1

De Python a Java: tu puente entre lenguajes

Si ya conoces Python (o cualquier otro lenguaje), esta guía traduce lo que sabes. Si no conoces ninguno, léela igual: entender por qué Java es diferente es parte de los valores de la semana (pensamiento crítico).
La gran pregunta de la semana
En Python escribes edad = 20 y listo. En Java escribes int edad = 20; dentro de una clase con un método main. ¿Por qué Java exige tanta estructura? Al final de esta guía tendrás argumentos para responder — y no solo "porque sí".

1. El mismo programa, en dos idiomas

🐍 Python☕ Java
 
print("Hola, mundo")
 
public class Main {
  public static void main(String[] args) {
    System.out.println("Hola, mundo");
  }
}
Sí: una línea contra cinco. Pero esas cuatro líneas "extra" no son burocracia — son un contrato. Sigue leyendo.

3. Los 5 tropiezos clásicos del pythonista en Java

1. Olvidar el ;
Cada instrucción termina en punto y coma. El error ';' expected será tu compañero la primera semana. Es normal.
2. True con mayúscula
Python: True/False. Java: true/false. El compilador dirá cannot find symbol.
3. Esperar que 7 / 2 dé 3.5
En Java, entero entre entero da entero: 3. Convierte primero: (double) 7 / 2.
4. Reasignar con otro tipo
En Python x = 5 y luego x = "hola" es válido. En Java no: la cajita int solo acepta enteros para siempre.
5. Usar and / or / not
Java usa símbolos: &&, ||, !. Escribir and es error de compilación.
Bonus: lo que NO cambia
La lógica es idéntica: variables, expresiones, precedencia de operadores, módulo, comparaciones. Aprender el segundo lenguaje siempre es más fácil que el primero.

4. ¿Por qué Java exige tipado fuerte? (pensamiento crítico)

🐍 Python (tipado dinámico)

  • El tipo se decide al ejecutar.
  • Escribir es rápido; ideal para scripts y prototipos.
  • Un error de tipos explota en producción, frente al usuario: 
    edad = "veinte"
edad + 1  # 💥 TypeError en ejecución

☕ Java (tipado fuerte y estático)

  • El tipo se verifica al compilar, antes de que nadie lo use.
  • El mismo error se detecta al instante: 
    int edad = "veinte";
// 💡 error: incompatible types
// (no llega ni a ejecutarse)
  • En proyectos grandes (como tu app web de este curso), el tipo es documentación viva: leer int edad te dice qué esperar sin adivinar.
Conclusión honesta
No es que Java sea "mejor" que Python: son herramientas para trabajos distintos. Python brilla en ciencia de datos y scripts; Java brilla en sistemas grandes, bancos y aplicaciones web empresariales (Spring Boot, semanas 7–18). La estructura que hoy parece exceso será tu red de seguridad cuando el proyecto tenga 30 archivos.

5. Ejercicios de traducción (Python → Java)

Traducción 1 — Ficha simple

Traduce este Python a Java y ejecútalo en Educative:

nombre = "Luis"
edad = 22
print("Hola", nombre)
print("El próximo año tendrás", edad + 1)
Solución 
public class Main {
  public static void main(String[] args) {
    String nombre = "Luis";
    int edad = 22;
    System.out.println("Hola " + nombre);
    System.out.println("El próximo año tendrás " + (edad + 1));
  }
}

Traducción 2 — Lógica booleana

Este es más tramposo (¡ojo con and y True!):

edad = 20
inscrito = True
puede = edad >= 18 and inscrito
print(puede)
print(not puede)
Solución 
public class Main {
  public static void main(String[] args) {
    int edad = 20;
    boolean inscrito = true;
    boolean puede = edad >= 18 && inscrito;
    System.out.println(puede);
    System.out.println(!puede);
  }
}

Traducción 3 — La división traicionera

En Python esto imprime 3.5. Logra que Java también imprima 3.5:

a = 7
b = 2
print(a / b)
Solución 
public class Main {
  public static void main(String[] args) {
    int a = 7;
    int b = 2;
    System.out.println((double) a / b); // 3.5
  }
}
Sin el casting imprime 3: división entera.

Traducción 4 — Reto inverso (Java → Python)

Ahora al revés: ¿qué imprime este Java? Escríbelo en Python mentalmente y verifica:

public class Main {
  public static void main(String[] args) {
    final int ANIO = 2026;
    int nacimiento = 2004;
    boolean mayor = (ANIO - nacimiento) >= 18;
    System.out.println("Mayor de edad: " + mayor);
  }
}
Solución
Imprime Mayor de edad: true (2026−2004 = 22 ≥ 18). En Python sería:
ANIO = 2026
nacimiento = 2004
mayor = (ANIO - nacimiento) >= 18
print("Mayor de edad:", mayor)  # True
