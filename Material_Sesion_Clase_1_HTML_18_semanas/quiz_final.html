¿Qué aprendiste hoy?

10 preguntas sobre estructura de un programa Java, tipos de datos, casting y operadores. Al terminar: Calificar → Exportar a PDF → subir al aula virtual.
1) ¿Cuál es el punto de entrada de todo programa Java, donde la JVM empieza a ejecutar?
✅ La JVM busca exactamente esa firma para arrancar.
Es el método main: la JVM busca exactamente public static void main(String[] args) para empezar.
2) ¿Qué tipo de dato usarías para guardar una estatura como 1.62?
✅ double guarda números con decimales.
Era double: int solo guarda enteros y boolean solo true/false.
3) ¿Qué imprime System.out.println(7 / 2);?
✅ División entera: int / int descarta los decimales.
Imprime 3: entre enteros la división es entera. Para 3.5 necesitas (double) 7 / 2.
4) ¿Qué valor queda en p? double precio = 9.99; int p = (int) precio;
✅ El casting explícito trunca: corta los decimales sin redondear.
Queda 9: el casting (int) trunca (corta) los decimales, no redondea.
5) ¿Cuál declaración de constante es correcta y sigue la convención?
✅ final + nombre en MAYÚSCULAS: constante correcta y bien nombrada.
La palabra clave es final y la convención pide MAYÚSCULAS: final double IVA = 0.12;
6) ¿Qué imprime System.out.println(17 % 5);?
✅ % devuelve el residuo: 17 = 5×3 + 2.
El operador % da el residuo de la división: 17 entre 5 cabe 3 veces y sobran 2.
7) ¿Cuál es la diferencia entre = y ==?
✅ Asignar vs comparar: el error clásico #1 ya no te atrapará.
= asigna (mete el valor en la caja); == compara (pregunta si son iguales y devuelve true/false).
8) Con int edad = 20; y boolean inscrito = false;, ¿qué imprime System.out.println(edad >= 18 && inscrito);?
✅ El && exige ambas: la edad cumple pero inscrito es false → resultado false.
Imprime false: && (Y) exige que ambas condiciones sean verdaderas, e inscrito es false.
9) ¿Qué imprime System.out.println(1 + 2 + "3");?
✅ Se evalúa de izquierda a derecha: 1+2=3 (números) y luego 3+"3" concatena → "33".
Imprime 33: primero 1+2=3 (suma de números) y después 3 + "3" ya es concatenación de texto.
10) El compilador marca: error: ';' expected en la línea 4. ¿Qué haces?
✅ Leer → ubicar → corregir → ejecutar. El error es información, no fracaso.
El mensaje ya te dice qué (;) y dónde (línea 4): corriges solo eso y ejecutas de nuevo.
