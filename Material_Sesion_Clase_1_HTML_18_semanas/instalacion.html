Tu entorno de desarrollo · VS Code + JDK + Git

Instala y configura tu computadora para programar

Guía súper detallada, paso a paso, con verificación de cada pieza, videos de apoyo, solución de problemas y un Plan B 100% online por si nada funciona hoy.
⏰ ¿Tengo que instalar todo esto HOY?
No. La Semana 1 se trabaja completa con el compilador online (Plan B, al final de esta página), sin instalar nada. Pero tu computadora debe estar lista a más tardar en la Semana 7 (Spring Boot exige JDK local). Nuestra recomendación: instala todo esta semana con calma, siguiendo esta guía; si algo se traba, tienes 6 semanas y la sección de troubleshooting para resolverlo sin presión.

Ruta de instalación (en este orden)

  1. Instalar el JDK (el motor de Java) — 10 min
  2. Instalar Visual Studio Code (el editor) — 5 min
  3. Instalar las extensiones de Java — 5 min
  4. Instalar Git (control de versiones) — 10 min
  1. Verificación final (checklist del semáforo) — 5 min
  2. Videos de apoyo (español)
  3. Troubleshooting: si algo falla
  4. Plan B: herramientas online (sin instalar nada)
Tiempo total estimado: 35–45 minutos con buena conexión. Hazlo con la compu conectada a la corriente.

Paso 1 — Instalar el JDK el motor

El JDK (Java Development Kit) es lo que realmente compila y ejecuta Java. VS Code sin JDK es como un carro sin motor. Usaremos Eclipse Temurin (de Adoptium): gratuito, oficial y el que recomienda el programa del curso.

1.1 ¿Qué versión? (2026)

  • El curso pide JDK 17 o superior. Las versiones LTS (soporte largo) vigentes en 2026 son 17, 21 y 25.
  • Recomendación del curso: JDK 21 (LTS) — es el equilibrio perfecto: moderno, estable y 100% compatible con Spring Boot (semanas 7–18). Si en la página te ofrece JDK 25 LTS como "Latest LTS Release", también sirve.

1.2 Descarga (Windows)

  1. Abre https://adoptium.net/es/temurin/releases/?version=21.
  2. La página detecta tu sistema. Verifica los selectores de arriba: Operating System: Windows · Architecture: x64 · Package Type: JDK (⚠ no JRE) · Version: 21 - LTS.
  3. Descarga el archivo que termina en .msi (es el instalador con asistente; el .zip es para usuarios avanzados).

1.3 Instalación (Windows) — la parte donde TODOS se equivocan

  1. Doble clic al .msi → Next.
  2. Acepta la licencia → Next.
  3. Llegarás a la pantalla "Custom Setup" (iconos de disco duro en un arbolito). Aquí está el truco:
    • Clic en el icono junto a "Set JAVA_HOME variable" → elige "Will be installed on local hard drive".
    • Verifica que "Add to PATH" ya esté marcado para instalarse (viene activado por defecto — no lo desactives).
  4. Next → Install → acepta el permiso de administrador → Finish.
⚠ El error #1: instalar el JDK y saltarse la casilla JAVA_HOME. El programa se instala, pero la terminal "no encuentra" Java. Si te pasó, no reinstales todavía: mira el troubleshooting T1.

1.4 Instalación (macOS)

  1. En la misma página elige Operating System: macOS y la arquitectura: aarch64 si tu Mac es Apple Silicon (M1/M2/M3/M4) o x64 si es Intel. (¿No sabes? Menú → "Acerca de este Mac" → si dice "Chip Apple M…", es aarch64.)
  2. Descarga el .pkg, doble clic y sigue el asistente (Continuar → Instalar). En Mac el instalador configura todo solo: no hay casillas especiales.

1.5 Verificar el JDK ✅

  1. Cierra cualquier terminal que tuvieras abierta (las terminales viejas no ven el PATH nuevo).
  2. Abre una terminal nueva:
    • Windows: tecla Windows → escribe powershell → Enter.
    • macOS: Cmd + espacio → escribe terminal → Enter.
  3. Escribe estos dos comandos (Enter después de cada uno):
java -version
javac -version
Salida esperada (los números menores pueden variar):
openjdk version "21.0.11" 2026-04-21 LTS
OpenJDK Runtime Environment Temurin-21.0.11+9 (build 21.0.11+9-LTS)
...
javac 21.0.11
✅ Checkpoint: si ambos comandos responden con "21" (o 17/25), tu motor Java está listo. Si aparece "no se reconoce como un comando…" → troubleshooting T1. Si java funciona pero javac no → instalaste el JRE en vez del JDK → T2.

Paso 2 — Instalar Visual Studio Code el editor

VS Code es el editor de código más usado del mundo: gratuito, liviano y el que usaremos las 18 semanas (Java hoy, Spring Boot después).

2.1 Descarga e instalación

  1. Abre https://code.visualstudio.com → botón azul "Download for Windows" (o Mac).
  2. Windows: ejecuta el instalador → acepta la licencia → deja la carpeta por defecto → en la pantalla "Tareas adicionales" marca TODAS las casillas, especialmente:
    • ✅ "Agregar la acción 'Abrir con Code' al menú contextual" (las dos).
    • ✅ "Agregar a PATH" (ya viene marcada — no la quites).
  3. macOS: se descarga un .zip → descomprime → arrastra Visual Studio Code.app a la carpeta Aplicaciones (importante: si lo dejas en Descargas, no se actualiza bien).
  4. Abre VS Code al terminar.

2.2 Conoce la interfaz (versión 2026) — para no perderte

VS Code cambió bastante en 2025–2026: si ves tutoriales viejos, algunas cosas se ven distintas. Lo esencial:

2.3 (Opcional) VS Code en español

  1. Abre Extensiones (Ctrl + Shift + X).
  2. Busca "Spanish Language Pack" (editor: Microsoft) → Install → botón Change Language and Restart.
  3. Nota: esta guía usa los nombres en inglés (los verás igual en la mayoría de tutoriales), pero el menú te quedará en español.

Paso 3 — Extensiones de Java para VS Code los superpoderes

Una sola instalación trae todo: el Extension Pack for Java de Microsoft (colores, autocompletado, botón Run, depurador, Maven y gestor de proyectos).

3.1 Instalar el Extension Pack for Java

  1. En VS Code, abre Extensiones: icono de 4 cuadritos en la Activity Bar o Ctrl + Shift + X.
  2. En el buscador escribe: Extension Pack for Java.
  3. Elige el que dice Microsoft como editor (icono con la taza de café ☕) → botón azul Install.
  4. Espera 1–3 minutos: instala 6 extensiones de golpe (verás que aparecen solas: Language Support for Java (Red Hat), Debugger for Java, Test Runner, Maven for Java, Project Manager…). No cierres VS Code mientras tanto.
  5. Si abajo a la derecha aparece un aviso tipo "Reload required" → clic en Reload.

3.2 Tu primer programa en VS Code (la prueba de fuego)

  1. Crea una carpeta en tus Documentos llamada programacion1 (sin tildes ni espacios — hazte ese favor desde hoy).
  2. En VS Code: File → Open Folder… → elige programacion1 → botón Seleccionar carpeta. Si pregunta "Do you trust the authors?" → Yes, I trust.
  3. En el Explorador (icono 📄), clic en el icono "New File" y nómbralo exactamente: HolaMundo.java
  4. Pega esto y guarda con Ctrl + S:
public class HolaMundo {
  public static void main(String[] args) {
    System.out.println("¡Mi entorno funciona!");
    System.out.println("Java listo para las 18 semanas.");
  }
}
  1. Verás un botoncito ▶ Run gris arriba del main (tarda unos segundos en aparecer la primera vez). Haz clic.
  2. Abajo se abre la Terminal y debe imprimir tus dos frases. 🎉
✅ Checkpoint: si ves "¡Mi entorno funciona!" en la terminal, tienes JDK + VS Code + extensiones correctamente conectados. Si el botón Run no aparece o da error → troubleshooting T4/T5.
💡 Tip para novatos: abre la Command Palette (Ctrl + Shift + P) y ejecuta "Java: Tips for Beginners" — la propia extensión te da un mini-tour guiado.

Paso 4 — Instalar Git control de versiones

Git guarda la historia de tu código (lo usaremos a fondo desde la Semana 3, con GitHub). Instalarlo hoy te deja listo y VS Code lo detecta automáticamente.

4.1 Windows — el instalador de las mil pantallas

  1. Descarga desde https://git-scm.com/download/win → "Click here to download" (64-bit, versión 2.5x).
  2. Ejecuta el instalador. Verás muchas pantallas: en casi todas la respuesta es Next sin tocar nada. Solo presta atención a estas 3:
PantallaQué elegirPor qué
"Choosing the default editor used by Git" Cambia el desplegable a "Use Visual Studio Code as Git's default editor" El default (Vim) es una trampa mortal para principiantes
"Adjusting the name of the initial branch" Marca "Override…" y deja el nombre main Es el estándar actual de GitHub (lo usaremos en la Semana 3)
"Adjusting your PATH environment" Deja la opción recomendada: "Git from the command line and also from 3rd-party software" Permite usar git desde PowerShell y desde VS Code
Todo lo demás (SSH, HTTPS, line endings "Checkout Windows-style", MinTTY, etc.): deja lo que viene marcado → Next → … → Install → Finish.

4.2 macOS

  1. Abre la Terminal y escribe git --version.
  2. Si no está instalado, macOS te ofrece instalar las Command Line Developer Tools → acepta (botón "Instalar") y espera. Eso es todo.

4.3 Preséntate con Git (una sola vez)

Git firma cada cambio con tu nombre. En una terminal nueva:

git config --global user.name "Tu Nombre Real"
git config --global user.email "tucorreo@ejemplo.com"
Usa el mismo correo con el que crearás tu cuenta de GitHub (Semana 3). Verifica con: git config --global --list

4.4 Verificar Git ✅

git --version
Salida esperada: git version 2.50.1.windows.1 (o similar). En VS Code, el icono de Source Control (tercer icono de la Activity Bar) dejará de ofrecer "Download Git".

Troubleshooting — si algo falla 🔧

Regla de oro: lee el mensaje de error completo antes de tocar nada (resiliencia ante el error, ¿recuerdas?). Busca tu síntoma:

  1. ¿Cerraste y reabriste la terminal? Las terminales abiertas antes de instalar no ven el PATH nuevo. Cierra TODAS (incluida la de VS Code) y abre una nueva. El 70% de los casos se resuelve aquí.
  2. Si persiste: el instalador no agregó Java al PATH. La vía rápida es reinstalar: vuelve a ejecutar el .msi de Temurin → elige Modify/Change → activa "Add to PATH" y "Set JAVA_HOME variable" → siguiente hasta terminar → terminal nueva → java -version.
  3. Vía manual (si no puedes reinstalar): tecla Windows → "variables de entorno" → Editar las variables de entorno del sistema → botón Variables de entorno → en "Variables del sistema" edita Path → Nuevo → pega la ruta del bin del JDK, normalmente C:\Program Files\Eclipse Adoptium\jdk-21.x.x-hotspot\bin → Aceptar todo → terminal nueva.

Instalaste el JRE (solo ejecuta) en lugar del JDK (ejecuta Y compila). Vuelve a adoptium.net, asegúrate de que el selector Package Type diga JDK, descarga el .msi e instálalo (puede convivir con lo anterior). Terminal nueva y verifica.

Windows: no marcaste "Agregar a PATH" al instalar. No es grave (puedes abrir VS Code desde el menú de inicio), pero para arreglarlo: abre VS Code → Ctrl + Shift + P → escribe "Shell Command"… si no aparece, reinstala VS Code marcando todas las casillas de "Tareas adicionales".
macOS: abre VS Code → Cmd + Shift + P → "Shell Command: Install 'code' command in PATH" → Enter → terminal nueva.

  1. Espera 1–2 minutos la primera vez: abajo a la derecha dice "Opening Java Projects / Importing…" mientras la extensión arranca.
  2. Verifica que el archivo termina en .java y que abriste una carpeta (File → Open Folder), no solo el archivo suelto.
  3. Revisa que el Extension Pack for Java esté instalado y habilitado (Ctrl + Shift + X → pestaña "Installed").
  4. Ctrl + Shift + P → "Java: Clean Java Language Server Workspace" → Reload and delete.
  5. Si dice que no encuentra un JDK: Ctrl + Shift + P → "Java: Configure Java Runtime" → debería listar tu Temurin 21; si no, reinicia VS Code después de instalar el JDK (el orden importa: primero JDK, luego VS Code abierto).

En Java, el nombre del archivo debe ser idéntico al de la clase, mayúsculas incluidas. holamundo.java ≠ HolaMundo.java. Renombra el archivo (clic derecho en el Explorador → Rename) o cambia el nombre de la clase para que coincidan.

  1. Causa típica: la carpeta del proyecto tiene tildes, ñ o espacios en la ruta (ej. C:\Users\José María\Programación). Mueve tu carpeta a una ruta simple: C:\dev\programacion1.
  2. Antivirus/Windows Defender bloqueando: agrega una exclusión para la carpeta de proyectos si tu equipo lo permite.
  3. Última bala: Ctrl + Shift + P → "Java: Clean Java Language Server Workspace".

Igual que T1: terminal nueva primero. Si persiste, reinstala Git y en la pantalla del PATH elige la opción del medio: "Git from the command line and also from 3rd-party software". Mientras tanto puedes usar la app "Git Bash" que se instaló en tu menú inicio (ahí git siempre funciona).

  1. VS Code tiene "User Installer" que NO pide administrador (es el de descarga por defecto) — ese sí podrás instalarlo.
  2. Para el JDK sin admin: descarga el .zip de Temurin (en vez del .msi), descomprímelo en C:\Users\TuUsuario\jdk21 y en VS Code configura Ctrl + Shift + P → "Java: Configure Java Runtime" → apunta a esa carpeta.
  3. Si nada de esto es posible: trabaja con el Plan B online — cubre perfectamente las semanas 1–6 y hablamos tu caso antes de la Semana 7.

Requisitos mínimos realistas: Windows 10/11 de 64 bits, 8 GB de RAM (con 4 GB se puede, cerrando todo lo demás) y ~3 GB libres de disco. Si tu equipo no llega: usa el Plan B online desde cualquier navegador (incluso desde una tablet) y escríbele al docente — buscamos solución juntos antes de la Semana 7 (laboratorio, equipo prestado, etc.). Nadie se queda fuera del curso por hardware.

Te confundiste de producto (le pasa a todos una vez 😄). Visual Studio (morado, gigante) es otro programa. Nosotros usamos Visual Studio Code (azul, ~100 MB): code.visualstudio.com. Cancela la otra descarga y listo.
¿Nada de esto resolvió tu caso?
Toma captura de pantalla del error completo (no foto borrosa del monitor 📵), anota qué paso estabas haciendo y publícalo en el foro del aula virtual o tráelo a la próxima sesión. Un error bien reportado se resuelve en minutos.

Mini-guía: tu primera vez en Educative (2 minutos)

  1. Abre educative.io/compilers/java (no pide registro).
  2. Verás un editor con una clase Main de ejemplo. Borra el contenido y pega el template del cheat sheet.
