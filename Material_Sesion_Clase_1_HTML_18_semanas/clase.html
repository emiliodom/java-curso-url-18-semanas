Demostración en vivo — pega y ejecuta en Educative:

public class Main {
  public static void main(String[] args) {
    System.out.println("Hola, mundo");
  }
}
  • public class Main: el contenedor. En Java todo vive dentro de una clase.
  • public static void main(String[] args): el punto de entrada. La JVM busca exactamente esta firma.
  • System.out.println(...): imprime y salta de línea. El ; cierra cada instrucción.
Demostración del error (¡a propósito!)
Borra el ; y ejecuta. Lee el error en voz alta: ';' expected. Mensaje clave de la semana: el compilador no te regaña, te ayuda — te dice qué y dónde. Resiliencia ante el error.

Metáfora: una variable es una cajita con etiqueta… pero en Java la cajita tiene forma: solo caben datos del tipo declarado.

public class Main {
  public static void main(String[] args) {
    String nombre = "Ana";        // texto (objeto, va con comillas dobles)
    int edad = 20;                // entero
    double estatura = 1.62;       // decimal
    char inicial = 'A';           // un solo carácter (comillas simples)
    boolean esEstudiante = true;  // verdadero o falso
    final int ANIO_ACTUAL = 2026; // constante: no puede cambiar

    int anioNacimiento = ANIO_ACTUAL - edad;

    System.out.println("Nombre: " + nombre);
    System.out.println("Edad: " + edad);
    System.out.println("Estatura: " + estatura);
    System.out.println("Inicial: " + inicial);
    System.out.println("¿Estudiante?: " + esEstudiante);
    System.out.println("Año de nacimiento aprox: " + anioNacimiento);
  }
}
  • Recorre tipo por tipo preguntando: "¿qué guardarías aquí?".
  • final = constante. Convención: MAYUSCULAS_CON_GUION. Demuestra el error al intentar reasignarla.
  • Convención de nombres: camelCase para variables (anioNacimiento), nunca empezar con número ni usar espacios.
  • El + con texto concatena (pega); con números suma.
Preguntas rápidas (piensa–escribe–comparte, 2 min)
  • ¿Qué pasa si guardo "veinte" en int edad? (Error de tipos: pruébalo.)
  • ¿Por qué char usa comillas simples y String dobles?
  • ¿Qué imprime System.out.println(1 + 2 + "3")? ¿Y "1" + 2 + 3? (¡Sorpresa! 33 vs 123.)
Pensamiento crítico (valor de la semana)
Pregunta al grupo: "En Python escribes edad = 20 sin declarar tipo. ¿Por qué Java te obliga a escribir int?" Respuesta guiada: el tipado fuerte detecta errores antes de ejecutar, hace el código más legible en proyectos grandes y permite a la JVM optimizar. Enlaza con la Guía del Programador de Python.

Metáfora: pasar agua entre recipientes. De cajita chica a grande no se pierde nada (implícito); de grande a chica puede derramarse (explícito: tú asumes el riesgo).

public class Main {
  public static void main(String[] args) {
    // Implícito (widening): int → double, sin pérdida
    int entero = 7;
    double decimal = entero;          // 7.0

    // Explícito (narrowing): double → int, se TRUNCA (no redondea)
    double precio = 9.99;
    int precioEntero = (int) precio;  // 9

    // División entera vs decimal (el clásico)
    int a = 7, b = 2;
    System.out.println(a / b);           // 3  (división entera)
    System.out.println((double) a / b);  // 3.5

    // Número → texto y texto → número
    String textoEdad = "20";
    int edad = Integer.parseInt(textoEdad);
    String edadComoTexto = String.valueOf(edad);

    System.out.println(decimal);
    System.out.println(precioEntero);
    System.out.println(edad + 1);          // 21
    System.out.println(edadComoTexto + 1); // "201"
  }
}
  • Insiste en el caso 7 / 2 = 3: es el error silencioso más común del curso.
  • (int) 9.99 da 9, no 10: truncar ≠ redondear.
  • Integer.parseInt aparecerá cuando leamos datos de formularios web (semana 10): siembra esa conexión.

public class Main {
  public static void main(String[] args) {
    int a = 17, b = 5;

    // Aritméticos
    System.out.println(a + b);  // 22
    System.out.println(a - b);  // 12
    System.out.println(a * b);  // 85
    System.out.println(a / b);  // 3  (entera)
    System.out.println(a % b);  // 2  (residuo/módulo)

    // Comparación (resultado: boolean)
    System.out.println(a > b);   // true
    System.out.println(a == b);  // false
    System.out.println(a != b);  // true

    // Lógicos: && (y), || (o), ! (no)
    int edad = 20;
    boolean tieneCarne = true;
    boolean puedeEntrar = edad >= 18 && tieneCarne;
    System.out.println(puedeEntrar);        // true
    System.out.println(!puedeEntrar);       // false
    System.out.println(edad < 18 || tieneCarne); // true
  }
}
  • Diferencia clave: = asigna, == compara. Escríbelo grande.
  • Las expresiones de comparación producen un valor (true/false) que se puede guardar en un boolean. La semana 2 usaremos esos booleanos en if/else.
  • % (módulo): pregunta "¿17 pizzas entre 5 personas, cuántas sobran?" → 2.
  • Tabla rápida &&/|| con ejemplos hablados: "mayor de edad Y con carné", "descuento si es estudiante O adulto mayor".
