Variables, tipos, casting y operadores (sin instalar nada)

Usar: Educative Java Compiler. Intenta primero sin ver la solución: primero la pista, luego la solución.
Abrir compilador
Template recomendado
Copia/pega esto al iniciar cada ejercicio. En Educative la clase debe llamarse Main.
public class Main {
  public static void main(String[] args) {
    // Tu código aquí
  }
}
Método de trabajo: escribe → ejecuta → lee la salida (o el error) → corrige → repite. Un cambio a la vez.

Ejercicio 1 — Hola Mundo + 3 líneas Calentamiento

Objetivo: comprobar que compila y entender la impresión en consola.

  1. Imprime Hola mundo.
  2. Imprime tu nombre completo.
  3. Imprime la frase: "Hoy empiezo a programar en Java".
  4. Extra: borra un ; a propósito, ejecuta y lee el error. Vuelve a ponerlo.
Solución sugerida 
public class Main {
  public static void main(String[] args) {
    System.out.println("Hola mundo");
    System.out.println("Ana López");
    System.out.println("Hoy empiezo a programar en Java");
  }
}

Ejercicio 2 — Mi ficha personal Variables

Objetivo: declarar un dato de cada tipo primitivo básico + String.

  1. Declara: String nombre, int edad, double estatura, char inicial, boolean leGustaProgramar.
  2. Asígnales tus datos reales.
  3. Imprime cada uno con una etiqueta, por ejemplo: Edad: 20.
Pista
Texto + variable se une con +: "Edad: " + edad. Recuerda: char con comillas simples 'A', String con dobles "Ana".
Solución sugerida 
public class Main {
  public static void main(String[] args) {
    String nombre = "Ana López";
    int edad = 20;
    double estatura = 1.62;
    char inicial = 'A';
    boolean leGustaProgramar = true;

    System.out.println("Nombre: " + nombre);
    System.out.println("Edad: " + edad);
    System.out.println("Estatura: " + estatura);
    System.out.println("Inicial: " + inicial);
    System.out.println("¿Le gusta programar?: " + leGustaProgramar);
  }
}

Ejercicio 3 — Calculadora de operadores Aritméticos

Objetivo: practicar +, -, *, /, % y descubrir la división entera.

  1. Define int a = 17; y int b = 5;.
  2. Antes de ejecutar, escribe en un comentario qué crees que imprime cada operación.
  3. Imprime suma, resta, multiplicación, división y residuo (%).
  4. ¿Te sorprendió a / b? Explica en un comentario por qué da 3 y no 3.4.
Pista
int / int siempre da int: se descarta la parte decimal. El residuo de 17 % 5 es lo que sobra tras repartir: 2.
Solución sugerida 
public class Main {
  public static void main(String[] args) {
    int a = 17;
    int b = 5;

    System.out.println("Suma: " + (a + b));        // 22
    System.out.println("Resta: " + (a - b));       // 12
    System.out.println("Multiplicación: " + (a * b)); // 85
    System.out.println("División: " + (a / b));    // 3 (entera)
    System.out.println("Residuo: " + (a % b));     // 2
  }
}
Nota: los paréntesis en (a + b) importan. Sin ellos, "Suma: " + a + b imprime Suma: 175 (concatena).

Ejercicio 4 — El cajero que trunca Casting

Objetivo: diferenciar conversión implícita y explícita, y ver la pérdida de datos.

  1. Declara double precio = 9.99;.
  2. Conviértelo a int con casting explícito e imprímelo. ¿Da 9 o 10?
  3. Declara int puntos = 250; y guárdalo en un double sin casting. ¿Por qué no exige (double)?
  4. Calcula 7 / 2 y luego (double) 7 / 2. Imprime ambos.
Pista
De chico a grande (int → double) Java convierte solo (widening). De grande a chico (double → int) debes escribir (int) y aceptar que se trunca.
Solución sugerida 
public class Main {
  public static void main(String[] args) {
    double precio = 9.99;
    int precioEntero = (int) precio;      // 9 (trunca, no redondea)
    System.out.println(precioEntero);

    int puntos = 250;
    double puntosDecimal = puntos;        // implícito: 250.0
    System.out.println(puntosDecimal);

    System.out.println(7 / 2);            // 3
    System.out.println((double) 7 / 2);   // 3.5
  }
}

Ejercicio 5 — ¿Verdadero o falso? Comparación y lógicos

Objetivo: construir expresiones booleanas con >, ==, &&, ||, !.

  1. Declara int edad = 20;, double promedio = 85.5;, boolean esBecado = false;.
  2. Crea e imprime estas expresiones booleanas (primero predice el resultado en un comentario):
    • edad >= 18
    • promedio > 90
    • edad >= 18 && promedio > 80
    • esBecado || promedio >= 85
    • !esBecado
  3. Guarda la tercera en una variable boolean puedeInscribirse e imprímela con etiqueta.
Solución sugerida 
public class Main {
  public static void main(String[] args) {
    int edad = 20;
    double promedio = 85.5;
    boolean esBecado = false;

    System.out.println(edad >= 18);                    // true
    System.out.println(promedio > 90);                 // false
    boolean puedeInscribirse = edad >= 18 && promedio > 80;
    System.out.println("¿Puede inscribirse?: " + puedeInscribirse); // true
    System.out.println(esBecado || promedio >= 85);    // true
    System.out.println(!esBecado);                     // true
  }
}

Ejercicio 6 — Constantes: la cuenta del restaurante final

Objetivo: usar constantes y combinar operadores en un cálculo real.

  1. Declara la constante final double PROPINA = 0.10; (10%).
  2. Declara double consumo = 185.50;.
  3. Calcula e imprime: la propina (consumo * PROPINA) y el total (consumo + propina).
  4. Intenta reasignar PROPINA = 0.15;, ejecuta, lee el error y luego borra esa línea.
  5. Extra: divide el total entre 4 amigos e imprime cuánto paga cada uno.
Solución sugerida 
public class Main {
  public static void main(String[] args) {
    final double PROPINA = 0.10;
    double consumo = 185.50;

    double propina = consumo * PROPINA;
    double total = consumo + propina;

    System.out.println("Propina: " + propina);   // 18.55
    System.out.println("Total: " + total);       // 204.05
    System.out.println("Cada uno paga: " + (total / 4)); // 51.0125
  }
}

Ejercicio 7 — Detective de errores Depuración

Objetivo: leer errores de compilación con calma (resiliencia ante el error). Este código tiene 4 errores. Cópialo, ejecútalo y corrígelos uno por uno.

public class Main {
  public static void main(String[] args) {
    int edad = "veinte";
    String nombre = 'Ana';
    double estatura = 1,75;
    System.out.println("Hola " + nombre)
  }
}
Pista
  • ¿"veinte" cabe en una cajita int?
  • ¿Qué comillas usa String?
  • ¿Java usa coma o punto para decimales?
  • ¿Qué le falta al final de la última instrucción?
Solución sugerida 
public class Main {
  public static void main(String[] args) {
    int edad = 20;                 // int guarda números, no texto
    String nombre = "Ana";         // String usa comillas dobles
    double estatura = 1.75;        // decimales con punto
    System.out.println("Hola " + nombre); // faltaba el ;
  }
}

Ejercicio 8 — Reto integrador: "Ficha de beneficiario" Integrador

Objetivo: integrar todo lo de hoy. Contexto del proyecto del curso: registrar personas de una entidad sin fines de lucro.

  1. Constante: final int ANIO_ACTUAL = 2026;
  2. Variables del beneficiario: nombre (String), año de nacimiento (int), estatura en metros (double), inscrito (boolean).
  3. Calcula la edad aproximada (ANIO_ACTUAL - anioNacimiento).
  4. Crea un boolean esMayorDeEdad con una comparación.
  5. Crea un boolean puedeRecibirPrograma: es mayor de edad Y está inscrito.
  6. Imprime una ficha completa, una línea por dato, con etiquetas claras.
Solución sugerida 
public class Main {
  public static void main(String[] args) {
    final int ANIO_ACTUAL = 2026;

    String nombre = "Carlos Pérez";
    int anioNacimiento = 2005;
    double estatura = 1.70;
    boolean inscrito = true;

    int edad = ANIO_ACTUAL - anioNacimiento;
    boolean esMayorDeEdad = edad >= 18;
    boolean puedeRecibirPrograma = esMayorDeEdad && inscrito;

    System.out.println("=== Ficha de beneficiario ===");
    System.out.println("Nombre: " + nombre);
    System.out.println("Edad: " + edad);
    System.out.println("Estatura: " + estatura);
    System.out.println("Inscrito: " + inscrito);
    System.out.println("Mayor de edad: " + esMayorDeEdad);
    System.out.println("Puede recibir el programa: " + puedeRecibirPrograma);
  }
}
